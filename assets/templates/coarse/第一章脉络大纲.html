计算机类型：用途、规模与算力决定系统形态

Computer Types
核心问题：为什么“计算机”会从嵌入式设备到超级计算机呈现出完全不同的形态？
分类依据
  • 按 size, cost, computational power, intended use 区分不同计算机。
  • 不同类型的设计目标不同：有的追求低功耗和特定控制，有的追求通用性，有的追求吞吐量或极限算力。
主要类型
  • Embedded computers：集成到更大设备中，自动监测和控制物理过程。
  • Personal computers：面向个人通用应用，如文档、计算、CAD 和娱乐。
  • Servers & Enterprise systems：通过网络服务大量用户，处理大规模请求和数据库。
  • Supercomputers & Grid computers：面向天气预报、工程设计、科学计算等高性能任务。
  • Cloud computing：通过互联网按需提供共享资源、软件和信息。

五大功能部件：数据进入、存储、处理、输出和控制

Functional Units
核心问题：一台计算机完成任务，最少需要哪些功能单元协同？
基本功能
  • 数据处理：对 machine instructions、numbers、characters 等数字信息进行处理。
  • 数据存储：保存程序和数据。
  • 数据传送：在计算机内部以及计算机与外部环境之间传递信息。
  • 控制：协调所有部件按程序要求工作。
五个基本功能单元
  • Input Unit：接受编码信息，如键盘、鼠标、扫描仪等。
  • Output Unit：把处理结果送到外部，如显示器、打印机等。
  • Memory：保存正在执行的程序与数据，并通过 cache、main memory、secondary storage 分层组织。
  • ALU：执行算术和逻辑运算。
  • Control Unit：发出控制信号，感知各部件状态，协调操作顺序。

程序执行：PC 指向下一条，IR 保存当前条

Operational Concepts
核心问题：程序如何从内存中的指令序列变成处理器内部的一连串动作？
指令与数据
  • 计算机处理的信息包括 instructions 和 data。
  • 指令显式规定信息传送、算术逻辑运算、I/O 交互等动作。
  • 程序执行就是不断重复 instruction fetch 和 instruction execution。
两个关键寄存器
  • PC：Program Counter，指向下一条将要执行的指令。
  • IR：Instruction Register，保存当前正在执行的指令。
  • 典型执行循环：Fetch Instruction -> Execute Instruction -> 判断是否 Halt。
中断的入口认识
  • Interrupt 是 I/O 设备向处理器提出服务请求。
  • 中断会打断普通取指-执行循环，引出后续 I/O 章节中的中断处理机制。

性能与世代：硬件技术推动计算机体系结构演进

Performance
核心问题：计算机性能为何不断提升？每一代技术替换解决了什么瓶颈？
性能影响因素
  • 最重要的性能指标是程序执行速度。
  • 性能受 instruction set、hardware、software 共同影响。
  • VLSI 让晶体管更小，开关更快，也使更多功能集成到单芯片成为可能。
计算机世代
  • Zero Generation：机械计算机。
  • First Generation：真空管，代表 ENIAC；冯·诺依曼提出 stored-program concept。
  • Second Generation：晶体管，磁芯存储和磁鼓存储广泛使用。
  • Third Generation：集成电路，大量晶体管集成到硅片。
  • Fourth Generation：LSI & VLSI，微处理器、个人计算机、嵌入式系统快速发展。
