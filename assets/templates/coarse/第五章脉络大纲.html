1

CPU 基本闭环：取指、译码、执行

Concepts
核心问题：处理器如何从内存取出指令，并把 ISA 中的语义变成硬件动作？
Processing Unit 的职责
  • Processing Unit 即 CPU / Instruction Set Processor。
  • 它负责从内存读取程序指令，解释指令，并执行指令指定的动作。
  • 最小闭环是 IR ← [[PC]] 与 PC ← [PC]+4。
为什么引入多级处理
  • 复杂操作可拆成多个 stage，每个 stage 在一个 clock cycle 内完成子任务。
  • 多级结构让连续指令能重叠执行，是 pipeline 的基础。
  • 流水线最希望各类指令经过相同数量的步骤，以保持节奏整齐。
2

五步执行模型：把不同指令塞进同一硬件节奏

5 Steps
核心问题：Load、Add、Store 行为不同，为什么还能共用五阶段框架？
统一五步
  • 1. Fetch instruction and increment PC。
  • 2. Decode instruction and read register operands。
  • 3. Execute / compute in ALU。
  • 4. Access memory if needed。
  • 5. Write result back if needed。
典型指令如何套入
  • Load R5, X(R7)：第 3 步算 EA，第 4 步读内存，第 5 步写 R5。
  • Add R3, R4, R5：第 3 步 ALU 运算，第 4 步 no action，第 5 步写 R3。
  • Store R6, X(R8)：第 3 步算 EA，第 4 步写内存，第 5 步 no action。
  • Branch 和 Call 通过第 3 步修改 PC 或保存返回地址。
3

五级数据通路：数据在寄存器、ALU、Memory 间流动

Datapath
核心问题：为了支持五步执行，硬件需要哪些部件和中间寄存器？
五级硬件组织
  • Stage 1：instruction fetch，把指令取入 IR。
  • Stage 2：decode + register read，把源寄存器读入 RA/RB。
  • Stage 3：ALU 计算，结果进入 RZ。
  • Stage 4：memory access，访存数据或写入数据通过 RY/RM 等路径传递。
  • Stage 5：write back，把最终结果写回 register file。
关键部件
  • Register file：需要两个读端口，同时读取两个源寄存器。
  • ALU：执行算术、逻辑、地址计算和比较。
  • Multiplexers：选择 ALU 输入、内存地址来源、写回结果来源。
  • Inter-stage registers：在阶段之间保存数据，保证下一周期继续处理。
4

控制信号：决定数据通路每一拍做什么

Control
核心问题：同一套硬件如何在不同指令、不同阶段执行不同动作？
控制信号的作用
  • 控制 multiplexer 选择，让数据走向正确部件。
  • 设置 ALU function，决定执行加、减、比较或逻辑操作。
  • 决定 PC、IR、register file、memory 等何时写入。
  • 控制 memory read/write，并在访存慢时等待 MFC。
Wait for MFC
  • Cache hit 时访存可能一个周期完成。
  • Cache miss 需要主存返回，Read/Write 步骤可能延长多个 clock cycles。
  • 取指阶段和 Load/Store 的访存阶段都可能需要 Wait for MFC。
  • PC 在扩展取指周期中只能递增一次，通常等 MFC 到来后再更新。
5

控制器实现：硬布线速度快，微程序灵活

Implementation
核心问题：控制信号是由固定逻辑生成，还是由“控制存储器里的程序”生成？
Hardwired Control
  • 由组合逻辑根据 step counter、IR、ALU result、MFC、interrupt requests 等生成控制信号。
  • Step counter 输出 T1 到 T5，表示当前处于五步中的哪一步。
  • 优点是速度快；缺点是成本高、修改和扩展不灵活。
Microprogramming
  • 把每个时钟周期需要的控制信号值存入 microinstruction / control word。
  • 一条机器指令由一段 microprogram / microroutine 实现。
  • 所有微程序存放在 control store 中。
  • 优点是灵活，适合复杂 CISC 指令；缺点是读取和执行微指令会带来额外延迟。
