1

整数表示：同一串 bit，解释方式决定数值

2-1
核心问题：机器如何表示正数、负数，以及为什么补码最适合做加减法？
三种有符号表示
  • Signed-Magnitude：最高位是符号，其余位表示绝对值，直观但运算麻烦。
  • One's Complement：负数由对应正数逐位取反得到，存在 +0 和 -0。
  • Two's Complement：负数由逐位取反再加 1 得到，只有一个 0，硬件加减最高效。
范围与扩展
  • n 位无符号整数范围为 0 到 2^n - 1。
  • n 位补码范围为 -2^(n-1) 到 2^(n-1) - 1，负数比正数多一个。
  • 补码扩展位长时做 sign extension：新高位填充原符号位。
2

加减法与加法器：正确性来自补码，速度瓶颈来自进位

2-2 到 2-4
核心问题：补码加法为什么能统一加减法？进位为什么决定硬件速度？
补码加减法规则
  • 补码加法：把符号位当作最高位一起加，忽略最高位 carry-out。
  • 减法 X - Y：先对 Y 求补码相反数，再执行 X + (-Y)。
  • 溢出只会发生在同号相加或异号相减时，结果符号异常即溢出。
从全加器到串行进位
  • 1-bit full adder 输入 xi、yi、ci，输出 si 和 ci+1。
  • n-bit ripple-carry adder 把多个全加器串接，低位进位逐级传到高位。
  • 缺点是延迟随位数线性增长：最高位和最终进位都要等待前面级联完成。
先行进位加法器
  • 引入 generate Gi 和 propagate Pi，提前计算各级进位。
  • Carry-lookahead adder 用更多逻辑门换取更短延迟。
  • 长字长加法器通常用 4-bit CLA 模块级联，受 fan-in / fan-out 限制。
3

乘除法算法：用移位、加减和寄存器状态完成复杂运算

2-5 到 2-7
核心问题：硬件如何把乘除法分解成可重复的加、减、移位步骤？
无符号乘法
  • 手算乘法按乘数每一位生成 partial product，n 位乘 n 位最多得到 2n 位积。
  • Array multiplier 并行生成部分积，速度快但硬件数量多。
  • Sequential multiplication 用一个加法器循环完成，硬件少但需要多个周期。
Booth 有符号乘法
  • 普通无符号乘法不能直接处理补码负数，Booth 算法统一处理正负乘数。
  • 观察 Q0 和 Q-1：01 加 M，10 减 M，00/11 只算术右移。
  • 连续 1 的乘数可减少加减次数，是 Booth 的效率来源。
整数除法
  • 恢复除法：试减为负则恢复 A，再置商位 0。
  • 不恢复除法：避免立即恢复，根据余数符号决定下一步加或减。
  • 课件强调没有像 Booth 那样简单统一的有符号除法算法。
4

浮点表示与运算：范围、精度和舍入的折中

2-8 到 2-9
核心问题：怎样用有限 bit 近似表示极大、极小和小数？运算后误差如何处理？
IEEE 754 表示
  • 浮点数形式为 (-1)^S x M x 2^e，机器中保存 S、偏置阶码 E、尾数字段。
  • 单精度：1 位符号、8 位阶码、23 位尾数，bias 为 127。
  • 双精度：1 位符号、11 位阶码、52 位尾数，bias 为 1023。
  • 特殊值包括正负零、无穷、NaN 和 denormal numbers。
浮点运算流程
  • 加减法先对阶：小阶数的尾数右移，结果阶码取较大者。
  • 乘法：阶码相加再减 bias，尾数相乘，符号位异或。
  • 除法：阶码相减再加 bias，尾数相除，符号位异或。
精度与异常
  • Guard bits 用于减少对阶和运算中的精度损失。
  • 截断方式包括 chopping、Von Neumann rounding 和 rounding。
  • 尾数溢出可通过规格化修正；阶码溢出才是真正范围溢出。
