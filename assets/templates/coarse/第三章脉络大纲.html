主存基础：地址、字、字节序与访问过程

3-1
核心问题：CPU 看到的内存是什么样子？一次读写需要哪些信号和寄存器？
地址空间与字节编址
  • 地址用于标识 memory words 或 byte locations，k 位地址可表示 2^k 个位置。
  • 现代系统通常 byte-addressable，连续地址对应连续字节。
  • Big-endian 低地址放高有效字节；little-endian 低地址放低有效字节。
读写操作
  • 读操作：CPU 把地址送入 MAR，设置 R/W=1，数据经 MDR 返回。
  • 写操作：CPU 把地址送入 MAR，把数据送入 MDR，设置 R/W=0。
  • 性能指标包括 access time、cycle time、transfer rate。

半导体存储器：SRAM 与 DRAM 的器件取舍

3-2 到 3-4
核心问题：为什么 Cache 常用 SRAM，而主存常用 DRAM？芯片如何扩展成更大存储器？
SRAM
  • 用触发器/锁存结构保存 bit，只要供电就能保持状态。
  • 优点是 fast；缺点是 low density、high cost。
  • 芯片组织题常考容量 M x N 与地址线、数据线数量。
DRAM
  • 用电容电荷保存 bit，读操作会刷新电容状态。
  • 优点是 high density、low cost；缺点是访问慢且需要 refresh。
  • SDRAM 与时钟同步，DDR 在时钟双沿传输数据，提高带宽。
存储器扩展
  • 位扩展：芯片位宽不足，多个芯片并行扩展数据位。
  • 字扩展：芯片字数不足，用译码器选择不同芯片组。
  • 字位同时扩展：总片数 = (目标字数/芯片字数) x (目标位宽/芯片位宽)。

存储层次：用局部性弥合速度、容量、成本矛盾

3-5
核心问题：为什么单一级存储器不够？层次结构靠什么有效？
理想存储器的不可能三角
  • 理想目标是 zero latency、infinite capacity、zero cost。
  • 现实中 bigger is slower，faster is more expensive。
  • 层次结构从寄存器、L1/L2 Cache、主存到磁盘逐级变大、变慢、变便宜。
局部性原理
  • 时间局部性：最近访问过的数据很可能很快再次访问。
  • 空间局部性：访问某项数据后，附近数据可能很快被访问。
  • 上层存储保留近期块，下层按块搬运连续数据。

Cache：命中率、映射方式、替换与写策略

3-6
核心问题：主存块放到 Cache 哪里？如何找到？Miss 时换谁？写入时如何保持一致？
Cache 基础
  • Cache 是小而快的 SRAM buffer，位于 CPU 和主存之间，由硬件自动管理。
  • Hit 表示目标块在 Cache 中；miss 表示必须访问主存装入块。
  • 平均访问时间由 hit rate、cache access time 和 miss penalty 共同决定。
三种映射
  • Direct mapping：每个主存块只能去一个 Cache line，简单便宜但冲突严重。
  • Fully associative：主存块可去任意 line，冲突少但比较硬件复杂。
  • Set associative：每个块映射到某个 set，可放入 set 内任一 line，折中最常用。
替换与写策略
  • 直接映射无替换选择；组相联/全相联常用 LRU、FIFO、Random。
  • Write through 同时更新 Cache 和主存，一致性简单但写流量大。
  • Write back 只先更新 Cache，并设置 dirty bit，替换时写回主存。
  • Write miss 可选 no-write allocate 或 write allocate。

虚拟内存与外存：把存储系统扩展到磁盘

3-7 到 3-8
核心问题：程序为什么能拥有比物理内存更大的地址空间？磁盘访问时间由什么构成？
虚拟内存与分页
  • Virtual memory 是抽象机制，用磁盘扩展物理内存的表观容量。
  • 虚拟地址分为 virtual page number 和 offset，页表把虚页映射到页框。
  • Page fault 类似 Cache miss，但代价远高，由操作系统从磁盘调页。
  • TLB 是页表项的小型高速缓存，TLB hit 可省掉一次页表访问。
磁盘组织与访问时间
  • 磁盘由盘片、盘面、磁道、扇区和柱面组织数据。
  • 同一柱面内切换磁道不需要移动磁头。
  • Disk access time = seek time + rotational latency + transfer time。
