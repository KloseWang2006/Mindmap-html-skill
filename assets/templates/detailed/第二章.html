Chapter 9: Arithmetic

依据第二章 2-1 至 2-9 课件整理的静态思维导图，按整数表示、加法器设计、乘除法算法、IEEE 754 与浮点运算展开。

1. 整数表示 (Integer Representation)
2-1 Page 51.1 n-bit 整数与无符号解释
一个 n 位二进制串 B = b(n-1)...b1b0 本身只是 bit；解释为无符号整数时，范围是 0 到 2^n - 1。
V(B) = b(n-1)2^(n-1) + ... + b1*2 + b0
复习抓手： 同一串 bit 按 unsigned、signed-magnitude、one's complement、two's complement 解释会得到不同数值。
2-1 Page 6 - 91.2 Signed-Magnitude Representation
最高位表示符号，0 为正、1 为负；其余 n-1 位表示绝对值。
特点：
• 正数形式与无符号表示相同
• 存在 +0 和 -0 两种零
• 表示范围为 -(2^(n-1)-1) 到 +(2^(n-1)-1)
缺点： 加减法要同时比较符号和绝对值大小，硬件控制复杂。
2-1 Page 10 - 111.3 One's Complement Representation
负数由对应正数逐位取反得到。
+18 = 00010010
-18 = 11101101
依然存在 +0 和 -0，因此可表示负数范围与原码类似。
2-1 Page 12 - 171.4 Two's Complement Representation
负数由“逐位取反 + 1”得到，是课件强调的最适合加减法的表示法。
+18 = 00010010
-18 = 11101110
范围: -2^(n-1) 到 +2^(n-1)-1
结论：
• 所有正数在三种有符号表示中相同
• 负数表示不同
• 补码只有一个 0，且加减法硬件最简单
2-1 Page 18 - 201.5 不同位长之间的转换
原码扩展：符号位移动到新的最左端，中间补 0。
补码扩展：符号扩展，新增高位填充原符号位。
8-bit -18 = 11101110
16-bit -18 = 1111111111101110
2. 补码加减法 (Addition and Subtraction of Signed Numbers)
2-2 Page 3 - 72.1 n-bit 补码加法与溢出
补码加法直接把符号位当作最高位参加运算，并忽略 MSB 的 carry-out。
X + Y: add n-bit representations, ignore carry-out from sign bit
Arithmetic Overflow： 运算结果超出 n 位补码可表示范围。充分必要条件是两个同号数相加得到异号结果。
例：4 位补码中 (+5)+(+4)=+9 超出范围 -8 到 +7，发生溢出。
2-2 Page 8 - 122.2 n-bit 补码减法
做 X - Y 时，先求 Y 的补码相反数，再执行加法。
X - Y = X + (-Y)
-Y = complement(Y) + 1
课件结论：n 位有符号数加减都可由 n 位二进制加法实现，符号位与其他位一起处理。
3. 加法器硬件 (Ripple-Carry Adder and Fast Adders)
2-3 Page 3 - 63.1 1-bit Full Adder
全加器输入 xi, yi, ci，输出和位 si 与进位 ci+1。
si = xi xor yi xor ci
ci+1 = xi*yi + xi*ci + yi*ci
2-3 Page 7 - 153.2 n-bit Ripple-Carry Adder
多个全加器串接形成 n 位加法器，进位从低位逐级 ripple 到高位。
s(n-1) delay = (2n-1)T
c(n) delay = 2nT
串行进位的核心瓶颈是等待进位传播，位数越长延迟越明显。
2-3 Page 16 - 173.3 Addition/Subtraction Logic Unit
用 XOR 控制 Y 输入是否取反，并设置初始进位 c0，即可在同一硬件中完成加法与减法。
Add/Sub = 0: Y unchanged, c0 = 0
Add/Sub = 1: Y complemented, c0 = 1
2-4 Page 3 - 113.4 Carry-Lookahead Addition
先行进位通过 generate / propagate 函数提前计算各级进位。
Gi = xi*yi
Pi = xi xor yi
ci+1 = Gi + Pi*ci
理论上所有进位可在 3 个门延迟后得到，所有和位可在 4 个门延迟后得到。
理解重点： CLA 用更复杂的逻辑网络换时间，本质是从“等待进位”变成“并行预测进位”。
2-4 Page 12 - 153.5 构造更长的 CLA
4-bit CLA 不能无限直接扩展，因为门的 fan-in / fan-out 有物理限制。
32-bit 加法器可由 8 个 4-bit CLA 级联构成，形成模块化折中。
4. 乘法算法 (Unsigned and Signed Multiplication)
2-5 Page 3 - 74.1 无符号乘法与 Array Multiplier
手算乘法按乘数的每一位决定是否加入被乘数的移位副本，形成 partial products。
n-bit x n-bit unsigned multiplication -> up to 2n-bit product
Array multiplier 并行生成和累加部分积，速度高但需要大量 AND 门和 full adders。
2-5 Page 8 - 114.2 Sequential Multiplication
顺序乘法使用一个 n 位加法器，通过 C, A, Q 寄存器循环完成。
流程：检查 Q0；若 Q0=1 则 A=A+M；随后整体右移；重复 n 次；最终积在 A 和 Q 中。
硬件资源少，但时间按循环次数增长，是面积与速度的折中。
2-6 Page 3 - 134.3 Booth Algorithm
Booth 算法直接处理补码有符号乘法，并可减少连续 1 产生的部分积数量。
Q0 Q-1 = 01: A = A + M
Q0 Q-1 = 10: A = A - M
Q0 Q-1 = 00 or 11: no add/sub
then arithmetic right shift A,Q,Q-1
优势： 正负乘数统一处理；当乘数有大段连续 1 时，加减次数更少。
Booth 的移位必须是算术右移，最高位保留符号。
5. 整数除法 (Integer Division)
2-7 Page 3 - 75.1 Restoring Division
恢复除法通过试减判断商位：若余数为负，说明试减失败，需要把除数加回去恢复。
核心循环： 左移 A,Q -> A=A-M -> 若 A<0 则 q0=0 且 A=A+M，否则 q0=1。
2-7 Page 8 - 115.2 Non-Restoring Division
不恢复除法避免每次失败后立即恢复，而是根据当前余数符号决定下一轮加 M 还是减 M。
若最终余数为负，最后再做一次恢复。
课件结论：没有与 Booth 乘法类似的简单有符号除法统一算法。
6. 浮点数表示 (Floating-Point Numbers and IEEE 754)
2-8 Page 3 - 76.1 Fixed-Point 的局限
定点数的小数点位置固定，适合整数或固定小数，但难以同时表示非常大的数和非常小的数。
浮点数的引入，本质上是把“有效数字”和“数量级”分开存储。
2-8 Page 8 - 116.2 Floating-Point Representation
浮点数类似科学计数法，二进制小数点位置可变，必须显式保存指数。
Value = (-1)^S x M x 2^e
E = e + bias
偏置表示避免负指数的补码形式被误认为很大的正指数。
2-8 Page 12 - 196.3 IEEE 754 Standard
Single Precision： 32 位，1 位符号，8 位 exponent，23 位 fraction，bias=127。
Double Precision： 64 位，1 位符号，11 位 exponent，52 位 fraction，bias=1023。
Special Values： ±0、±infinity、NaN、denormal numbers。
-0.6875 = -1.011 x 2^-1
S=1, E=126=01111110, M=011000...
7. 浮点运算 (Floating-Point Number Arithmetic)
2-9 Page 3 - 67.1 加减乘除规则
Add/Subtract： 先对阶，右移较小阶码数的尾数；阶码取较大者；尾数相加减；最后规格化。
Multiply： 阶码相加并减 bias，尾数相乘，确定符号，必要时规格化。
Divide： 阶码相减并加 bias，尾数相除，确定符号，必要时规格化。
2-9 Page 7 - 157.2 Guard Bits 与截断/舍入
浮点尾数字段长度有限，运算后多出的位必须处理。
Guard bits： 保留额外低位，减少对阶右移和后续运算导致的精度损失。
Chopping： 直接丢弃多余位，实现简单但有偏。
Von Neumann rounding： 若被丢弃部分含 1，则保留部分最低位置 1。
Rounding： 按最近可表示数舍入，课件用 0舍1入法举例。
2-9 Page 16 - 217.3 规格化与溢出
规格化保证尾数形如 1.M，移位尾数时要同步调整阶码。
Mantissa Overflow： 同号尾数相加产生最高位进位，可右移尾数并增加阶码修正。
Exponent Overflow： 阶码超过范围，才是真正的浮点溢出。
判断题重点：尾数溢出不等于真实溢出；阶码溢出才表示结果超出可表示范围。
