Chapter 5: Basic Processing Unit

依据第五章课件整理的静态思维导图，采用与第四章一致的大纲式模板，围绕 CPU/Processing Unit、5-step instruction execution、datapath、control signals、hardwired control 与 microprogramming 展开。

1. 基本概念 Some Fundamental Concepts
Page 31.1 Processing Unit / CPU 是什么
Processing Unit 也就是 Instruction Set Processor / Central Processing Unit (CPU)，它的职责是从内存中读取程序指令并执行。
CPU 的基本工作流：
• fetch one instruction at a time
• decode / interpret the instruction
• carry out the actions specified by the instruction
本章主线： 第四章讲“指令长什么样”，第五章讲“这些指令在处理器内部到底怎么一步步跑起来”。
Page 4 - 71.2 Instruction Execution / 指令执行的最小闭环
课件先给出最基本的 fetch-execute cycle。任何指令执行，都可以先抽象成“取指 + 更新 PC + 执行”三件事。
IR ← [[PC]]
PC ← [PC] + 4
若内存按字节编址、每条指令 4 bytes，则取到当前指令后，PC 自动加 4 指向下一条。
若一条指令占多个 memory word，则 step 1 和 step 2 可能要重复多次，直到完整取到该 instruction。
绝大多数指令最终都是下列基本操作的某种组合：读 memory、读 registers、做 arithmetic/logic、把结果写回 register 或 memory。
Page 8 - 101.3 Multi-stage 与 Pipeline / 多级与流水线
数据处理系统往往不是“一坨组合逻辑一次做完”，而是拆成多个 stage，每个 stage 在一个 clock cycle 内完成自己的子任务。
为什么要分 stage：
• 让处理从一个阶段流向下一个阶段
• successive instructions 可以重叠执行
• 这就是 pipeline / 流水线 的基本思想
关键认识： pipelining 最希望“所有指令都走相同数量的步骤”，这样各 stage 的节奏最整齐，硬件利用率最高。
2. 五步执行模型 5-step Instruction Execution
Page 11 - 152.1 为什么统一成 five steps
课件用一个典型的 RISC-style processor 说明：为了让流水线组织清晰，尽量把各种指令都映射到统一的 5-step sequence 中。
统一的五步框架：
1. Fetch instruction and increment PC
2. Decode instruction and read register operands
3. Execute / compute in ALU
4. Access memory if needed
5. Write result back if needed
理解重点： 并不是每条指令都“天然需要 5 步”，而是为了统一 hardware stages，某些指令会在个别步骤里 No action。
Page 122.2 Memory access instruction: Load
课件以 Load R5, X(R7) 为例，说明一个带 indexed addressing 的内存读指令如何走完五步。
1. Fetch the instruction and increment PC
2. Decode and read R7
3. Compute EA = X + [R7]
4. Read memory source operand
5. Write operand into destination register R5
这条指令完整体现了：取指、译码、地址计算、访存、写回 五个 stage。
Page 13 - 142.3 Computational instruction 与 Store instruction
Add R3, R4, R5： 不需要访问 memory operand，因此第 4 步没有动作，第 5 步把结果写回 R3。
Store R6, X(R8)： 需要把寄存器内容写到 memory，因此第 4 步执行 memory write，第 5 步没有动作。
统一五步的意义： Load、ALU、Store 三类典型指令虽然细节不同，但都能被塞进同一个五阶段硬件框架里。
3. 五级硬件组织 5-stage Hardware Components
Page 16 - 213.1 5-stage implementation of a RISC processor
课件把处理器硬件组织成五级：每个 clock cycle，instruction 从一个 stage 推进到下一个 stage。
五级硬件 stages：
• Stage 1: instruction fetch
• Stage 2: instruction decode + register read
• Stage 3: ALU computation
• Stage 4: memory access if needed
• Stage 5: write result into destination register
和第二节对应关系要记牢： 5-step execution 是“动作顺序”，5-stage hardware 是“这些动作由哪些硬件阶段承接”。
Page 17 - 193.2 Register File 与 ALU / 寄存器堆与算术逻辑单元
2-port register file： 为了同时读两个源寄存器，需要两个独立输出端 A 和 B。
寄存器堆除了读端口外，还有写数据输入 C 和目标寄存器地址输入，用于 write-back。
ALU： 负责 arithmetic operations 和 logic operations，如 add, subtract, AND, OR, XOR。
对立即数指令，ALU 的一个 source operand 可以来自 IR 中的 immediate field，而不是 RB。
Page 22 - 293.3 Datapath / 数据通路的关键寄存器与多路选择器
课件把 stage 2 到 stage 5 的数据流称为 datapath，并强调需要 inter-stage registers 在各阶段之间传递结果。
关键中间寄存器：
• RA, RB：保存从 register file 读出的源操作数
• RZ：保存 ALU stage 的结果
• RM：保存待写入 memory 的数据
• RY：保存 memory stage 或 ALU stage 最终选择出的结果
关键多路器 Mux：
• MuxB：选择 RB 或 immediate value 送入 ALU
• MuxY：选择 memory data 或 [RZ] 进入 RY
• MuxMA：选择 PC 或 RZ 作为 memory address source
Instruction fetch section：
• PC 提供 instruction address
• instruction address generator 负责更新 PC
• 它还能生成 branch address 与 subroutine address
Immediate block： 将 immediate operand 按 instruction 规定扩展为 32 bits。
4. 取指与执行实例 Instruction Fetch and Execution Steps
Page 30 - 354.1 典型指令的五步细化
本节非常适合复习，因为课件把多种典型 instruction 都展开成了可直接背诵的 5-step 微动作序列。
Add R3, R4, R5：
1. Memory address ← [PC], Read memory, IR ← Memory data, PC ← [PC] + 4
2. Decode, RA ← [R4], RB ← [R5]
3. RZ ← [RA] + [RB]
4. RY ← [RZ]
5. R3 ← [RY]
Load R5, X(R7)： Step 3 计算 effective address，Step 4 读 memory，Step 5 写回寄存器。
Store R6, X(R8)： Step 3 计算地址并把待写数据送到 RM，Step 4 执行 memory write，Step 5 no action。
Unconditional Branch： Step 3 直接更新 PC 为 [PC] + Branch offset。
Conditional Branch： Step 3 比较 [RA] 与 [RB]，若满足条件再更新 PC。
Subroutine Call with indirection： 先保存返回地址，再把 PC 改为目标地址，最后把返回地址写入 LINK register。
Page 36 - 374.2 Wait for Memory / Wait for MFC
真实系统里，memory access 不一定一个时钟周期就结束。若数据在 cache 中，通常 1 cycle 即可；若需要访问主存，则可能要等待多个 cycles。
处理器-内存接口会在操作完成时发出 MFC = Memory Function Completed 信号。
Memory address ← [PC], Read memory, Wait for MFC, IR ← Memory data, PC ← [PC] + 4
要点： 不仅 step 1 取指时需要 Wait for MFC，Load 和 Store 的 step 4 访存时同样需要。
5. 控制信号 Control Signals
Page 38 - 455.1 Control Signals 控制什么
处理器各硬件部件的动作并不是“自动发生”的，而是由 control signals 精确控制。
控制信号要完成的事包括：
• 控制 multiplexer selection，引导 data flow
• 指定 ALU function
• 决定何时把数据写入 PC、IR、register file、memory
课件分别讨论了：register file control signals、ALU control signals、result selection、memory and IR control signals、instruction address generator control。
一句话理解： datapath 负责“路怎么走”，control signals 负责“什么时候开哪条路、做哪种运算、写到哪里去”。
Page 425.2 Memory access 与 MFC control
cache hit 时 memory access 可以在 1 clock cycle 内结束；cache miss 时就必须等待更久，因此控制电路必须能识别 memory function 是否已经完成。
MFC 的作用就是让 control unit 知道“当前这一步可以结束了吗”。
6. 硬布线控制 Hardwired Control
Page 46 - 526.1 Hardwired control 的基本思想
生成 control signals 有两种基本思路：Hardwired control 和 Microprogramming。Hardwired control 是用组合逻辑电路直接生成控制信号。
Hardwired control 考虑的输入：
• step counter 的状态
• IR 中的 opcode / addressing mode 信息
• ALU result / condition inputs
• external inputs，如 MFC、interrupt requests
instruction decoder 会把 IR 解释成相应的 INSi 输出；step counter 用 T1 ~ T5 指出当前执行到哪一步。
C_i = f (INS_m, T_n, E_k, F_j)
也就是说，每个 control signal 都是 instruction type、timing step 和条件输入的组合逻辑函数。
Page 49 - 526.2 典型逻辑表达式与 memory delay 处理
Step 1 取指时： 需要设置 MA_select 选 PC、启动 MEM_read、在 MFC 到来时使能 IR_enable，并更新 PC。
示例逻辑：
RF_write = T5 · (ALU + Load + Call)
它说明寄存器写回既与“当前是第几步”有关，也与“当前是哪类指令”有关。
处理 memory delay：
Counter_enable = WMFC + MFC
当需要等待 memory function complete 时，step counter 暂停，当前步骤会被延长到多个 clock cycles。
PC 只应在恰当时刻更新一次：
PC_enable = T1 · MFC + T3 · BR
这一页很爱出选择题： 控制信号不是单看 opcode 就能定，还必须结合 timing signals 与外部状态信号一起判断。
7. 微程序控制 Microprogramming
Page 53 - 567.1 Microinstruction, Microprogram, Control Store
Microprogramming 是一种“软件化”的控制信号生成方法：把每个时钟周期需要输出的控制信号值存进一条 microinstruction / microinstruction (control word) 中。
核心概念：
• microinstruction / control word：一个时钟周期所需的控制信号位模式
• microprogram / microroutine：实现一条 machine instruction 所需的一串 microinstructions
• control store：存放这些 microprograms 的专用存储区
• μPC：记录当前 microinstruction 所在位置
优点： 灵活，特别适合实现更复杂的 CISC instructions。
缺点： 读取并执行 microinstructions 会引入额外延迟，不利于 high-performance processors。
课件还提到可以把 mutually exclusive signals 分组放入不同 fields，以减少 control word 的位数，但这样会增加一点 decoding logic。
