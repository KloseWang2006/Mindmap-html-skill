1. 存储系统基本概念 (Basic Concepts of Memory Systems)
3-1 Page 51.1 Internal Memory 与 External Memory
计算机系统配备层次化存储子系统。Internal memory 位于系统内部、可被处理器直接访问；external memory 通常通过 I/O 系统间接访问。
全章主线： 越靠近 CPU，速度越快、容量越小、单位成本越高；越远离 CPU，容量越大、速度越慢、单位成本越低。
3-1 Page 6 - 91.2 Memory Locations and Addresses
主存由大量 bit cell 构成，但通常以固定大小的 word 或 byte 为基本访问单位。
Address Space： k 位地址通常表示从 0 到 2^k - 1 的连续地址。
Byte-addressable Memory： 连续地址对应连续字节，不为每个 bit 分配独立地址。
3-1 Page 10 - 111.3 Byte Ordering
Big-endian：低地址保存更高有效字节。
Little-endian：低地址保存更低有效字节。
典型题：32 位值 0x30A79847 存入地址 0x1000，判断某地址处字节值。
3-1 Page 12 - 151.4 Main Memory Operations
Read/Load： 处理器把地址送入 MAR，将 R/W 置 1，内存把数据送到 MDR。
Write/Store： 处理器把地址送入 MAR、数据送入 MDR，将 R/W 置 0，写入指定位置。
3-1 Page 16 - 281.5 存储器特性
Physical types： semiconductor、magnetic surface、optical。
Capacity： word size 与 number of words 共同决定容量。
Access methods： sequential、random、direct、associative。
Performance： access time、cycle time、transfer rate。
cycle time = access time + recovery/additional time before next access
2. 半导体 SRAM (Semiconductor Static RAM)
3-2 Page 3 - 92.1 SRAM Cell
SRAM 用传统 flip-flop / latch 逻辑保存二进制值，word line 未选中时保持状态。
Read： 选中 word line，通过 bit lines 和 sense circuitry 读出状态。
Write： 外部写入电路强制 bit lines 到目标值，改变锁存状态。
优缺点： SRAM fast，但 low density、high cost；因此常用于 Cache。
3-2 Page 10 - 152.2 SRAM Chip Organization
课件用 16 x 8 与 1K x 1 展示 cell array、address decoder、sense/write circuitry 与控制线。
8K x 16 SRAM -> address lines = 13, data lines = 16
容量题抓住两个量：字数决定地址线，位宽决定数据线。
3. 半导体 DRAM (Semiconductor Dynamic RAM)
3-3 Page 3 - 63.1 DRAM Cell
DRAM 用电容 C 上的电荷表示 1 或 0，读操作经 sense amplifier 放大并刷新。
Read 1： 电荷高于阈值，sense amplifier 驱动 bit line 到 1，并重新充电。
Read 0： 电荷低于阈值，bit line 被拉低，保持无电荷状态。
优缺点： DRAM high density、low cost，但 longer access time、leaky、needs refresh。
3-3 Page 7 - 93.2 DRAM Chip Organization 与 Fast Page Mode
DRAM 为减少引脚数量，常把行地址与列地址分时复用；选中一行后，sense amplifier 也可作为 latch。
Fast page mode 利用同一行中连续访问，避免重复选择行。
3-3 Page 10 - 173.3 SDRAM 与 DDR
SDRAM 操作与时钟同步，可通过 mode register 选择 burst operation。
Latency： 第一个数据返回前等待的时钟周期数。
Bandwidth： 单位时间可传输的数据量。
DDR SDRAM： 在时钟上升沿和下降沿都传输数据，长 burst 情况下带宽近似翻倍。
4. 更大存储器结构 (Structure of Larger Memories)
3-4 Page 3 - 94.1 静态存储系统扩展
位扩展法： 芯片每个存储单元位数小于目标字长，多个芯片并行提供不同 bit 位。
字扩展法： 芯片字数不足，用高位地址经译码器选择芯片组。
字位同时扩展： 字数与位宽都不足，芯片形成二维阵列。
目标 M x N bit, 芯片 l x k bit
total chips = (M/l) x (N/k)
3-4 Page 10 - 144.2 DRAM Modules
大型 DRAM 系统与静态存储系统组织思想相似，实际封装为插在主板插槽上的 memory module。
SIMM：较旧的单列直插内存模块。
DIMM：现代常见封装，可一次提供 64 位数据。
5. 存储层次 (Memory Hierarchy)
3-5 Page 3 - 75.1 为什么需要层次结构
理想存储器要求零延迟、无限容量、零成本，但现实中容量、速度、成本互相冲突。
层次结构目标：用多个层级制造“fast and large”的错觉。
典型层次：registers -> L1 cache -> L2 cache -> main memory -> disk。
3-5 Page 8 - 135.2 Buffer 与 Locality
上层小而快的存储器作为下层大而慢存储器中部分数据的 buffer。
Temporal locality： 最近访问过的 item 很可能很快再次被访问。
Spatial locality： 访问一个 item 后，附近 item 可能很快被访问。
Cache 和虚拟内存都依赖局部性，只是块大小和 miss 代价不同。

Q1: Where can a block be placed? - block placement
Q2: How is a block found? - block identification
Q3: Which block should be replaced on a miss? - replacement
Q4: What happens on a write? - write strategy
6. Cache Memories
3-6 Page 3 - 106.1 Cache 的目的、组织、Hit/Miss
Cache 是由 SRAM 构成的小而快的硬件管理缓冲区，位于 CPU 和主存之间，保存主存中频繁访问的 blocks。
Hit： 请求数据已经在 Cache 中，直接返回。
Miss： Cache 中没有目标块，需要访问主存并装入包含目标字的 block。
CPU 不需要显式知道 Cache 存在，Cache control circuitry 自动判断 hit/miss。
3-6 Page 11 - 136.2 Cache Design Issues
Mapping scheme：主存块能放到 Cache 哪里？
Block identification：如何判断目标块是否在 Cache 中？
Replacement algorithm：miss 时替换哪一块？
Write policy：写命中和写缺失如何处理？
3-6 Page 14 - 216.3 Direct Mapping
每个主存块只能映射到唯一 Cache line。
i = j mod m
i: cache line number, j: main memory block number, m: cache lines
地址结构： Tag + Line + Word/Byte offset。
优点： simple、easy to implement、inexpensive。
缺点： fixed location for given block，容易发生 conflict miss。
3-6 Page 22 - 256.4 Fully Associative Mapping
主存块可装入 Cache 任意 line，因此地址只需 tag 和 word/byte offset。
优点： 放置自由度最大，冲突最少。
缺点： 需要同时比较所有 tag，硬件复杂且成本高。
3-6 Page 26 - 346.5 Set Associative Mapping 与 Valid Bit
组相联是直接映射和全相联的折中：主存块映射到某个 set，可放入该 set 内任意 line。
m = v x k
i = j mod v
address = Tag + Set + Word/Byte offset
k-way set associative： 每个 set 包含 k 条 line。
Valid bit： 标记 Cache block 中数据是否有效，刚上电时 Cache 内容无效。
3-6 Page 35 - 466.6 Replacement Algorithms 与映射例题
直接映射 Cache 没有替换选择；全相联和组相联需要替换算法。
LRU：替换最长时间未访问的块。
FIFO：替换最早进入的块。
Random：随机替换，实现简单。
直接映射中使用随机替换没有意义，因为 incoming block 只有一个可放位置。
3-6 Page 47 - 506.7 Write Policies
Write Through： 写命中时同时更新 Cache 和主存；一致性简单，但所有写都访问主存。
Write Back： 只更新 Cache 并设置 dirty/modified bit；替换时再写回主存。
No-Write Allocate： 写缺失时直接写主存，不把块调入 Cache。
Write Allocate： 写缺失时把包含目标字的块装入 Cache，再写 Cache。
3-6 Page 51 - 566.8 Hit Rate、Average Access Time 与 Multi-level Cache
命中率是成功在 Cache 中访问的数据次数占所有访问次数的比例。
H = Nc / (Nc + Nm)
tave = H * tc + (1 - H) * tm
现代系统通常使用多级 Cache：L1 常在 CPU core 内，可能分为 instruction cache 和 data cache；L2/L3 容量更大但更慢。
7. 虚拟内存与分页 (Virtual Memory and Paging)
3-7 Page 3 - 127.1 Virtual Memory 的动机与收益
虚拟内存用磁盘扩展物理内存的表观大小，并让每个程序看到连续、独立的地址空间。
Before VM： overlaying 需要程序员手工划分程序段，维护成本高。
Benefits： large address space、simplify relocation、support protection and sharing、ease multiprogramming。
Implementation： paging、segmentation、segmentation with paging。
3-7 Page 13 - 187.2 Paging 基本结构与地址转换
虚拟内存和主存都被划分为固定大小块：虚拟内存中叫 page，主存中叫 page frame。
Virtual address = Virtual Page Number + Offset
Physical address = Page Frame Number + Offset
Page table： PTE 数组，用虚拟页号索引，得到页框号或磁盘位置。
PTE 字段： page frame number、valid/presence bit、protection/use/dirty 等控制信息。
3-7 Page 19 - 237.3 Page Fault、TLB 与页面替换
Page hit： 访问的虚拟页已在主存。
Page fault： 虚拟页不在主存，需要 OS 从磁盘调入，代价远高于 Cache miss。
TLB： Translation Lookaside Buffer，是 page table 的小型高速缓存；TLB hit 可省掉一次内存页表访问。
Replacement： LRU、FIFO 等策略通常由 OS 软件实现。
3-7 Page 24 - 267.4 Page Size 的取舍
大页面优势： 页表更小，TLB 覆盖范围更大，磁盘批量传输效率更高。
大页面缺点： 内部碎片更严重，可能装入更多不会用到的数据。
Offset 位数由页面大小决定；例如 2MB page 需要 21 位 offset。
8. 辅助存储 (Secondary Storage)
3-8 Page 3 - 88.1 磁盘机械结构
磁盘由一个或多个盘片安装在共同 spindle 上，盘片以恒定速度旋转。
Fixed head： 每条 track 一个读写头。
Movable head： 每个 surface 一个读写头，磁臂移动定位到目标 track。
Winchester disk 采用密封组件，磁头靠近盘面高速飞行。
3-8 Page 9 - 148.2 数据组织：Track、Sector、Cylinder
Track： 每个盘面上的同心圆磁道。
Sector： track 被划分出的扇区，通常含 header、data、ECC、gap。
Cylinder： 所有盘面上半径相同的 track 集合。
访问同一柱面上的所有磁道不需要移动读写头，这是选择题常考点。
3-8 Page 15 - 168.3 Disk Access Time
读取磁盘块的总时间由定位磁头、等待扇区旋转到磁头下方、实际传输数据三部分构成。
Access Time = Seek Time + Rotational Latency + Transfer Time
Seek time： 磁头移动到目标 track 的时间。
Rotational latency： 等待目标 sector 转到磁头下方的时间。
Transfer time： 磁头定位后实际传输数据的时间。
