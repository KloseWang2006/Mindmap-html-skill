Chapter 2: Instruction Set Architecture

依据第四章课件整理的静态思维导图，采用分节大纲式排版，突出指令系统、指令格式、寻址方式、子程序与 RISC/CISC 的复习主线。

1. 指令与指令顺序 (Instruction and Instruction Sequencing)
Page 3 - 41.1 本节总览：程序到底由哪些基本操作构成
课件先给出 4 类最基本的指令行为，它们共同构成程序执行的骨架：
• memory 和 processor registers 之间的数据传送
• arithmetic and logic operations on data
• program sequencing and control
• I/O transfers
复习抓手： 后面所有复杂格式和寻址方式，本质上都在回答两件事：“要做什么操作” 和 “操作数到底从哪里来”。
Page 5 - 61.2 Register Transfer Notation, RTN
RTN 用来在硬件层级精确描述数据传送与寄存器操作，是理解 ISA 的最底层语言。
基本记号：
• 寄存器：R0, R5
• I/O 寄存器：DATAIN, OUTSTATUS
• 内存单元：LOC, PLACE, A, VAR2
• [X] 表示位置 X 中的内容，← 表示把右边的值送往左边目标
R2 ← [LOC]
R4 ← [R2] + [R3]
右侧表达式始终表示“值”，左侧始终表示“目标位置”。这让我们能把“取数、运算、写回”三个动作分得非常清楚。
Page 7 - 81.3 Assembly-Language Notation
RTN 适合描述硬件动作，但要写程序还需要更接近机器指令的表示法，于是就有了 assembly language。
核心特点：
• 一条指令同时给出 operation 和 operands
• 课件示例直接使用英文单词：Load, Store, Add
• 商用处理器通常使用 mnemonic，如 LD, ST, ADD
Load R2, LOC
Add R4, R2, R3
理解重点： RTN 更像“动作语义”，Assembly 更像“程序写法”；两者描述的是同一件事，只是抽象层级不同。
Page 9 - 141.4 RISC 与 CISC 的指令观
RISC： one-word instructions，指令少而简单，算术/逻辑操作要求操作数在寄存器中，典型代表有 ARM、MIPS、RISC-V。
CISC： multi-word instructions，允许直接对内存操作数进行处理，指令数量更多，单条指令常需要更多周期，代表是 x86。
这里是全章主线之一： 课件后面讲的指令长度、操作数位置、寻址方式，其实都和这两种设计哲学直接相连。
RISC 的 load/store architecture：
只有 Load 和 Store 访问内存；真正的算术逻辑运算在寄存器之间完成。
C = A + B

Load R2, A
Load R3, B
Add R4, R2, R3
Store R4, C
Page 15 - 171.5 Instruction Execution, Straight-Line Sequencing 与 Branching
程序默认按顺序执行。若字长为 32 位且存储器按字节编址，那么 PC 会按 i, i + 4, i + 8 ... 这种方式递增。
执行过程两阶段：
• Instruction Fetch
• Instruction Execute
[PC] = i
[PC] = i + 4
...
Branching： 把新的地址装入 PC，从而打破 straight-line sequencing，实现循环与条件控制。
• branch target 是跳转后的目标地址
• conditional branch 需要比较寄存器内容或依赖 condition codes
• 典型写法：Branch_if_[R4]>[R5] Loop
2. 指令格式 (Instruction Formats)
Page 18 - 202.1 Machine Instruction 的基本组成
Machine instruction 是处理器电路真正执行的指令；Instruction set 则是处理器全部可执行指令的集合。
Elements of a Machine Instruction：
• Opcode：规定做什么操作
• source operand reference：源操作数在哪里
• result operand reference：结果应该写到哪里
源操作数和结果操作数可以来自三类位置：main/virtual memory、processor register、I/O device。
Instruction = Opcode field + Address field
Page 21 - 222.2 常见地址数字段格式
Zero-address instruction： 不显式给出地址，常与栈式结构配合。
One-address instruction： 默认另一个操作数在累加器 AC 中。
[AC] OP [A] → AC
Two-address instruction：
[A1] OP [A2] → A1
A1 既是目的位置，也是参与运算的一个源操作数。
Three-address instruction：
[A2] OP [A3] → A1
结果位置可以与两个源操作数完全分离，表达能力最强，但位数开销也更大。
Page 23 - 302.3 Instruction Length 与 Opcode Format
Instruction length：
• fixed length：所有指令等长，译码更直接
• variable length：不同指令长度不同，编码更灵活
Fixed-length opcode： 若 opcode 占 k 位，则最多表示 2^k 种操作；若地址字段占 n 位，则可寻址 2^n 个单元。
Variable-length opcode / Expanding Opcode： 在总指令长度固定时，让 opcode 和 address field 彼此“借位”，从而同时支持不同地址数的指令。
课件经典设计题： 16-bit instruction、4-bit operand address 时，示例设计结果是：
• 15 条 three-address instructions
• 14 条 two-address instructions
• 31 条 one-address instructions
• 16 条 zero-address instructions
本质理解： 地址字段越多，留给 opcode 的位数越少；地址字段越少，opcode 就能更长，容纳更多同类指令编码。
3. 寻址方式 (Addressing Modes)
Page 31 - 323.1 寻址方式到底解决什么问题
Addressing mode 研究的是：指令中地址字段的位应该如何解释，才能确定操作数的真实位置。
这里的核心符号是 EA，即 Effective Address，表示最终真正访问的有效地址。
Page 333.2 Immediate mode
操作数直接写在指令内部。
Add R4, R6, #200
用途： 定义常量、初始化变量。
优点： 除取指外不需要额外访存。
缺点： 只能提供常量，而且常量大小受地址字段位数限制。
Page 34 - 353.3 Absolute mode / Direct mode
操作数位于某个内存单元，而该地址直接写在指令里。
EA = A
示例： Load R2, NUM1
用途： 访问编译时地址已知的全局变量。
优点： 只需一次访存，不需要额外计算。
缺点： 总是指向固定位置，可用地址空间有限。
Page 36 - 373.4 Indirect mode
指令中的地址字段并不是最终操作数地址，而是“存放操作数地址的那个地址”。
EA = [A]
这种方式在 CISC 风格机器中更常见，也可扩展成多级间接寻址：
EA = [...[A]...]
优点： 地址空间非常大，若字长为 N，则可扩展到 2^N 级别。
缺点： 先取地址、再取数值，至少需要两次访存。
课件还提到 indirect flag I：若 I = 0，当前字给出 EA；若 I = 1，则继续再做一层间接。
Page 38 - 393.5 Register mode
操作数就在处理器寄存器中。
EA = Ri
示例： Add R4, R2, R3
优点： 地址字段短，无需访问内存，速度快。
缺点： 地址空间非常有限，因为寄存器数量有限。
Page 40 - 433.6 Register Indirect mode
寄存器里存放的不是数据本身，而是某个内存操作数的地址。
EA = [Ri]
示例： Load R2, (R5)
它适合遍历列表、数组或连续数据区，因为只要更新寄存器中的指针值，就能访问下一项。
优势： 不必在指令中放完整内存地址，既保留了访问内存的能力，又减少了指令位宽压力。
Page 44 - 493.7 Indexed mode
有效地址由“寄存器中的基准值 + 指令中的偏移量”共同形成。
EA = X + [Ri]
其中 X 是指令中给出的常量偏移，Ri 常作为 index register；生成 EA 时寄存器本身不变。
典型用途： 访问数组、记录、二维表等“相对某参考点”组织的数据结构。
变体 1：Base with index
EA = [Ri] + [Rj]
变体 2：Base with index and offset
EA = [Ri] + [Rj] + X
Page 503.8 Relative mode
Relative mode 实际上是把 PC 当作 index register 来使用。
EA = [PC] + X
其中 X 是有符号偏移量。
用途：
• 访问相对当前位置的数据
• 在 branch 指令中表示 target address
复习理解： Relative addressing 的优势在于不用把完整目标地址硬编码进指令，只需记录“离当前位置偏多少”。
4. 子程序 (Subroutines)
Page 51 - 544.1 子程序的作用与返回机制
如果某个任务会被多次执行，例如数学函数计算、排序、公共处理逻辑，就没有必要在主程序中反复复制同一段代码，而应把它封装成 subroutine。
调用： 使用 Call 指令跳入子程序。
返回： 子程序最后通过 Return 指令返回调用点之后的位置。
Subroutine Linkage 是关键： 同一个子程序可能从不同位置被调用，所以系统必须保存“调用之后应该回到哪里”。
最简单的方案： 使用 link register 保存返回地址。
• 执行 Call 时，先把更新后的 PC 保存到 link register
• 然后 branch 到子程序入口
• 执行 Return 时，再跳回 link register 中保存的地址
5. RISC and CISC Styles 总结对照
Page 55 - 585.1 两种指令系统风格的核心差异
RISC characteristics：
• simple addressing modes
• all instructions fitting in a single word
• fewer total instructions
• arithmetic/logic operations on registers
• load/store architecture for data transfers
• more instructions executed per program
CISC characteristics：
• more complex addressing modes
• instructions spanning more than one word
• more instructions in the instruction set
• arithmetic/logic operations on memory
• memory-to-memory data transfers
• fewer instructions executed per program
一句话收束全章： RISC 用“更多但更简单”的指令换取更快、更规则的硬件实现；CISC 用“更复杂但更强大”的单条指令换取更紧凑的程序表达。
6. 课后题提示 (Homework Hint)
Page 59 - 606.1 Homework 的真正考点
最后两页作业都围绕 expanding opcode 设计 展开，本质上考的是“在总位数固定时，如何在不同地址数指令之间分配 opcode 空间”。
做题思路：
• 先由 instruction length 和 address field size 算出每类指令最多能留多少位给 opcode
• 先满足高地址数指令的编码需求，再把剩余编码空间扩展给低地址数指令
• 最后统计 each type of instruction 的 maximum number
